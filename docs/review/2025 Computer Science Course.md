## 📘 5 月 19 日：AP CSP（Computer Science Principles）体验课

### 课程重点（详细说明）：

1. **课程介绍与考试说明**

   - 向学生讲解 AP CSP 课程定位：适合零基础或初学者的计算机科学入门课程
   - 展示课程六大主题（数据、算法、网络、编程、计算影响、创造性项目）
   - 介绍考试结构：选择题 + 书面写作任务（Create Performance Task）
   - 展示往年样题，帮助学生了解评分标准与作答风格

2. **什么是“计算思维”？**

   - 用现实生活中的例子（如地图导航、快递排序、社交平台推荐）说明 CS 在生活中的广泛应用
   - 强调“不是先学语言，而是先学思维”
   - 引导学生理解：程序=有逻辑的流程，而不是神秘的代码堆

3. **图形化编程实践：用 Scratch 或 Code.org 实现小游戏**

   - 任务目标：设计一个简单角色互动或小游戏（如“点击角色变颜色”、“迷宫逃脱”等）
   - 学生通过拖拽指令块实现流程控制、条件判断、事件响应
   - 老师边带领边展示“事件驱动式编程”背后的逻辑思想

4. **学生展示与反馈**

   - 邀请部分学生演示成果
   - 引导学生讲述“他们遇到的问题是如何解决的”，培养计算思维中的“分解与调试”概念

## 📙 5 月 20 日：AP CSA（Computer Science A）体验课

### 课程重点（详细说明）：

1. **课程结构与考试特点分析**

   - 介绍 AP CSA 使用 Java 语言，课程内容包括变量、流程控制、数组、类与对象、继承等
   - 展示考试结构：选择题（MC）+ 编程题（FRQ），难点在于逻辑严谨性与代码规范
   - 分析学生可能遇到的困难：语法错误、逻辑混乱、调试能力不足

2. **编程基础讲解：变量、条件、循环**

   - 以“猜数字小游戏”为例，逐步讲解 Java 变量定义、if 判断、while 循环等
   - 通过代码演示，让学生理解“程序是如何一步步执行的”
   - 引导学生动手试写代码片段，感受从无到有的过程

3. **FRQ 题型解析与调试演练**

   - 演示一题典型的 FRQ（如编写一个方法处理数组、设计一个类）
   - 带学生一步步分析题目、提取信息、设计方法
   - 教授如何发现并修复常见 bug（拼写错误、数组越界、返回类型不一致等）

4. **实践分享与扩展建议**

   - 展示实际项目代码片段（可用你自己开发的小工具、游戏 DEMO 等）
   - 强调“编程不仅仅是考试，而是未来多领域必备能力”
   - 引导学生后续如何系统地提升编程能力（推荐学习路径、资源平台）

## 📗 5 月 20 日：中学生趣味编程入门课（45 分钟）

### 课程重点（详细说明）：

1. **什么是编程？用生活语言解释“程序”**

   - 举例说明：煎鸡蛋=顺序执行；打游戏=判断与循环；点外卖=算法与路径优化
   - 用贴近生活的比喻，让学生理解“其实每天都在用程序思维”

2. **快速上手：Scratch 可视化编程体验**

   - 任务示例：“点击角色变颜色”、“按方向键移动小猫”
   - 强调“编程就像搭积木”，人人都能做到
   - 教会学生基本概念：事件、动作、条件、循环

3. **成果展示与成就感反馈**

   - 每位学生完成一个小动画或小游戏
   - 鼓励上台展示作品，营造参与感和归属感
   - 总结：“你今天已经写出了一个程序”
